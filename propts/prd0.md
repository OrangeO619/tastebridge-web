1. 文档信息
项目	内容
产品名称	TasteBridge
产品定位	以地图为核心的探店记忆共享平台
目标平台	Web（移动优先），后续可考虑小程序/App
技术栈	React 18 + TypeScript + Next.js 14 + Tailwind CSS + Mapbox GL JS + Supabase + Vercel


探店爱好者：喜欢发掘新店，希望系统化记录体验

美食记录者：吃饭必拍照，想给食物打标签、写感受

小团体组织者：和几个朋友组成"美食小分队"，共享好店

核心痛点：

信息分散：位置在 Maps，口味在点评App，感受在备忘录，回忆时东拼西凑

重复踩坑：忘了某家店好不好吃，过段时间又去一次

分享负担：想推荐给朋友，要发定位+发点评+发照片，操作繁琐

社交压力：不想在群里频繁刷屏打扰朋友，但又想共享

2.2 核心价值主张
从"分散记录"到"地图上的记忆博物馆"：

用户到店 → 一键标记点位 → 记录喜爱值 + 文字/图片 → 形成个人情绪地图 → 可选择共享给好友 → 好友可补充自己的喜爱值 → 地图上叠加多重视角

2.3 设计风格关键词
清新、食欲、轻社交、地图优先

3. 核心功能模块
3.1 地图视图（核心）
功能点	描述	交互细节与逻辑
基础地图展示	加载底图，显示用户已标记的餐饮店点位	使用 Mapbox GL JS，支持缩放/平移；点位以自定义图标显示（可区分 种草/拔草/待去）
点位聚合	缩放级别低时，密集点位自动聚合为数字标记	点击聚合数字展开为多个单独点位；聚合逻辑在客户端实现
点位详情卡片	点击地图标记，从底部弹出卡片	卡片包含：店名、地址、喜爱值、缩略图、简短评价、上次到访时间
喜爱值视觉化	点位图标颜色或大小随喜爱值变化	高喜爱值用暖色大图标，低喜爱值用冷色小图标；一目了然
地图筛选	按喜爱值范围、标签、到访时间筛选显示的点位	筛选器悬浮在地图角落，实时更新地图显示
3.2 点位标记与记录
功能点	描述	交互细节与逻辑
添加点位	两种方式：搜索添加 / 长按地图添加	- 搜索：调用Mapbox Geocoding API
- 长按：在地图任意位置长按，拖拽调整精确位置
店铺信息	自动填充/手动填写	搜索模式自动填充店名、地址；手动模式支持自定义
喜爱值评分	用户对店铺的综合喜好程度	设计为5星；可附加一句话短评
情绪标签	可选预置标签或自定义	预置：🥰 惊喜 / 😌 舒服 / 🤔 一般 / 😤 踩雷；支持自定义Emoji+文字
AI标签推荐	根据用户笔记和评分，自动推荐相关标签	在标签选择区显示"AI推荐"标识，用户可一键添加
语音记录	支持语音输入替代打字	点击麦克风按钮说话，Whisper API实时转录为文字，可编辑后保存
图文记录	上传照片 + 详细笔记	支持多图上传；笔记支持 Markdown 或富文本
到访时间	自动记录当前时间，支持修改	便于日后回忆"那年今日"
3.3 喜爱值系统
喜爱值是 TasteBridge 的核心情感指标，采用多维评分设计：

typescript
// 喜爱值系统定义
type PrefValue = {
  overall: number;      // 综合评分 1-5
  dimensions?: {        // 可选的多维度
    taste: number;      // 口味 1-5
    ambiance: number;   // 氛围 1-5
    service: number;    // 服务 1-5
    value: number;      // 性价比 1-5
  };
  emoji?: string;       // 用户自定义情绪Emoji
  tags: string[];       // 自定义标签 + AI推荐标签
  lastVisit: string;    // 最后到访时间
  visitCount: number;   // 到访次数（自动累计）
}
喜爱值显示规则：

点位图标颜色 = 根据 overall 映射（5分深红、4分橙红、3分黄色、2分浅绿、1分灰色）

点位图标大小 = 根据 visitCount 调整（到访越多，图标略大）

点击点位，卡片上展示详细维度评分（如果有）

3.4 AI智能搜索与推荐
功能点	描述	交互细节与逻辑
语义化搜索	支持自然语言搜索	用户输入如"适合约会的餐厅""带爸妈去的店"，AI解析意图并返回匹配结果
相似店铺推荐	在店铺详情页推荐相似店铺	基于当前店铺特征和用户历史喜爱值，推荐相似店铺并附推荐理由
智能发现	在首页增加"猜你喜欢"模块	根据用户历史记录，推荐可能感兴趣的新店
3.5 社交共享与协作
功能点	描述	交互细节与逻辑
地图共享	用户可选择将整个地图或特定点位共享给好友	生成共享链接 / 邀请特定用户
好友协作模式	好友被邀请后，可在同一地图上添加自己的喜爱值	同一店铺可有多人的喜爱值，地图上以"堆叠"方式显示，互不覆盖
异步提醒	好友添加新点位时，非实时通知	采用摘要式通知（如"本周你的美食地图新增3个点位"），避免频繁打扰  
AI共同回忆	系统定期生成"你们一起探索的美食地图"回顾	AI聚合多人的喜爱记录，生成共同回忆文案
协作逻辑：

text
店铺 A 被 User1 标记，喜爱值 4.5
User2（被邀请好友）访问同一地图，可在店铺 A 添加自己的喜爱值 3.0

地图显示方式：
- 默认显示所有好友的平均喜爱值（可配置）
- 点击店铺，展开显示每个好友的个人评分和备注
- 支持筛选"只看好友推荐"或"只看我的标记"
3.6 个人中心与历史回顾
功能点	描述
我的足迹	地图上高亮显示所有我到过的店，支持时间轴浏览
喜爱值统计	我评分的分布、我最爱的菜系/区域
探店日历	按月份展示到访记录
月度总结	每月1日自动生成，包含本月探店数量、平均喜爱值、最爱店铺TOP3、情绪关键词
年度总结	每年1月自动生成，包含全年总览、年度最佳店铺、菜系偏好、情绪词云、成就徽章
待去清单	种草但还没去的店，可标记"想去"

5. AI功能详细设计
5.1 标签推荐
项	说明
触发时机	用户完成评分和笔记后，在标签选择区显示
输入	用户笔记、评分、菜系
输出	3-5个推荐标签（如"服务贴心""上菜快""适合拍照"）
交互	标签旁显示"AI推荐"标识，点击即可添加
模型	GPT-4o-mini
5.2 语音转文字
项	说明
触发时机	用户点击笔记输入框旁的麦克风图标
交互	长按录音，松手自动转录；转录结果实时显示在输入框，可编辑
模型	Whisper API
优势	支持中英文混合，识别准确率高

语义化搜索：

用户可输入自然语言，如"适合带爸妈去的餐厅""安静适合办公的咖啡馆"

AI解析意图，转化为结构化查询条件

6. 技术难点与解决方案
难点	挑战描述	解决方案
地图性能	大量点位同时渲染导致卡顿	视野内按需加载 + 聚合显示 + Mapbox GL的Canvas渲染优化
位置去重	同一家店被多人重复创建	调用地理编码服务时保存placeId；创建新店前按坐标范围搜索是否已存在
协作冲突	多人同时添加喜爱记录	按用户隔离存储，不修改他人记录；显示时合并展示
AI成本控制	免费版API有调用限制	缓存总结结果；标签推荐使用mini模型；限制单日调用频率
隐私边界	用户不想暴露所有足迹	共享时可选择"仅共享部分标签/评分"；支持隐藏特定点位和特定时间段内的足迹
地图服务调用次数限制：防抖节流 后端缓存

7. 交互设计要点
地图即主页：打开应用就是地图，所有操作围绕地图展开

喜爱值可视化：点位颜色/大小传递情感信息，一眼看懂

轻量记录：添加记录步骤控制在3步以内（长按 → 评分 → 可选照片 → 完成）

AI融入自然：标签推荐以"推荐"标识出现，语音输入作为备选，总结以精美卡片呈现

社交不打扰：采用摘要通知，避免实时刷屏

情绪Emoji系统：让记录更有温度

8. 差异化亮点总结
对比对象	他们的做法	TasteBridge 的做法
大众点评/美团	公共评分，陌生人为主，需跳转地图	个人喜爱值为主，地图即详情页
Google Maps 保存	仅保存位置，无深度记录	评分+笔记+照片+情绪标签一体化
小红书收藏	碎片化，难整理，无地理聚合	所有记录自动落位到地图，可视化沉淀
微信群分享	实时刷屏，打扰他人	异步协作，不打扰，可随时查看
传统记录工具	无AI辅助	AI标签推荐、语音输入、智能总结、语义搜索
